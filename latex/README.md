# LaTeX — CTUthesis Conventions Quick Reference

Project uses the custom `CTUthesis` class (`latex/CTUthesis.cls`).
Compile with `latexmk` (see `latex/latexmkrc`). Backend: `pdflatex` + `biber`.

---

## Acronyms — `acro` package

Declarations in `texparts/references/acro.tex` (tag `zkr`) and `texparts/references/acro_variables.tex` (tag `vel`).

| Macro | When to use |
|-------|-------------|
| `\ac{ID}` | Auto use — on first use expands like `\acf` "long form (SHORT)"; subsequent uses expand like `\acs` |
| `\acs{ID}` | Repeated use — short form only |
| `\acp{ID}` | Plural variant |
| `\acl{ID}` | Long form only |
| `\aclp{ID}` | Long form plural |
| `\acf{ID}` | Forced full form (long + short) |
| `\Ac{ID}` | Capitalised first word (sentence-initial) |
| `\acuse{ID}` | Register use without printing (e.g. variable in math) |

- `make-links=true` — `\ac{}` calls link to the list entry.
- Lists printed by `\printacronyms[include=zkr]` and `\printacronyms[include=vel]`.
- Sorted alphabetically by long form (`\CTUuseLongSortForAcronyms`).

## Citations — BibLaTeX / `\cite{}`

- **Use `\cite{key}`** — `\parencite{}` is NOT loaded.
- Place before the full stop, after a tilde: `…výsledek.~\cite{key}` or `…dohodě~\cite{key}.`
- Multiple consecutive: `\cite{key1}\cite{key2}` (no space).
- Bibliography files declared via `\addbibresource{}` in `latex/main.tex` (currently: `bibliography.bib`). Search those files for citation keys.

#### Adding a new abbreviation

Add to `latex/texparts/references/acro.tex`:
```latex
\DeclareAcronym{ID}{
	short		= SHORT,
	long 		= long form,
	foreign 	= English expansion,   % omit if same as long
	long-plural-form = plural form,  % omit if regular
	single = false,
	single-style = long-short,
	tag			= zkr
	}
```

Add a math variable to `latex/texparts/references/acro_variables.tex`:
```latex
\DeclareVariable{var-id}{\ensuremath{X}}{description [unit]}
\DeclareIndex{var-id}{\ensuremath{X_0}}{\ensuremath{0}}{description}
```

## Cross-references — `\label` / `\ref`

| Prefix | Floating type |
|--------|--------------|
| `ch:` | chapter |
| `sec:` | section / subsection |
| `p:` | part |
| `fig:` | figure |
| `tab:` | table |
| `eqn:` | equation |

- Float label inside `\caption{}`: `\caption{Popis.\label{fig:name}}`
- Inline: `v~kapitole~\ref{ch:uvod}`, `na obrázku~\ref{fig:name}`, `v~rovnici~\ref{eqn:name}`
- **Custom link text**: `\ref{}` always typesets the number. To hyperlink arbitrary text to a label use `\hyperref[label]{custom text}`:
  ```latex
  v~\hyperref[p:teorie]{teoretické části}
  ```

---

## Non-breaking spaces `~`

Always use `~` (tilde):
- After single-letter prepositions: `v~`, `k~`, `s~`, `z~`, `u~`, `o~`, `a~`
- Before `\cite{}` and `\ref{}`: `kapitole~\ref{…}`, `dohodě~\cite{…}`
- Between numeral and word/unit: `32~\%`, `tři~pilíře`

## Paragraph endings

End each body paragraph with `\par`:
```latex
Věta ukončující odstavec.\par
```
Not used inside list items.

## Units — `siunitx`

```latex
\SI{75}{\percent}   \SI{40}{\degreeCelsius}   \SI{5}{\kelvin\per\metre}
```
Bare decimal in text: `11{,}4~\%` (comma as decimal separator via `{,}`).

## Figures

### Standard (PDF) figure
```latex
\begin{figure}[htbp]
  \centering
  \includegraphics[width=\columnwidth]{../python/figures/filename}
  \caption{Popis obrázku.\label{fig:filename}}
\end{figure}
```

### PGF (LaTeX-native) figure — `\inputpgffigure{name}`
PGF figures are generated by Python scripts using `apply_style_pgf()` + `savefig_pgf()` +
`save_figure_tex_pgf(strings={...})`. The text in the figure is rendered by LaTeX, so
`\ac{}`, `\SI{}{}`, `\acs{geo-XX}` etc. work natively.

Include a PGF figure in commentary or main.tex with:
```latex
\inputpgffigure{figure_name}
```
This macro (defined in `CTUthesis.cls`) resolves to:
```latex
\input{texparts/figures/figure_name}   % if the .pgf file exists
```
or shows a yellow warning box if the script hasn't been run yet.

The file `texparts/figures/figure_name.tex` is **git-tracked and hand-editable**.
It contains:
- `\def\strNameCaption{...}%` — editable caption (with cite, acro macros)
- `\def\strNameXxx{...}%` — editable annotation strings used inside the figure
- `\begin{figure}...\end{figure}` — the full figure environment

To reset strings to data defaults: delete the strings file and re-run the Python script.

**File roles (PGF workflow):**

| File | Written by | Git | Purpose |
|------|-----------|-----|---------|
| `python/figures/<name>.pgf` | Python (every run) | ❌ ignored | PGF TeX fragment with macro references |
| `python/figures/_shared/img-<hash>.png` | Python (dedup cache) | ❌ ignored | Shared companion rasters (typically colourbars) reused across PGFs |
| `texparts/figures/<name>.tex` | Python (first run only) | ✅ tracked | Editable `\def`s + full figure env |
| `texparts/python/<name>.tex` | Python (every run) | ❌ ignored | `\inputpgffigure{name}` — backward compat |

## List of Software (`list_software` / `Seznam použitého softwaru`)

The thesis features a mandatory declaration section listing the software libraries, compilers, operating systems, and packages used to compile the text and analyze datasets.

- **File location**: `texparts/references/list_software.tex`.
- **Inclusion**: Handled by the class template `CTUthesis.cls` during compilation (suppress via document class option `nosoftware`).
- **Structure**:
  - Organizes tools into categories: Editor & Project Management, TeX Live (engines, compilers, and packages), Bibliography Management, Operating Environment, and Python Core Data Stack.
  - Custom command `\swsubitems{...}` formats nested packages with tight styling, suitable for the strict page budget.
- **Maintenance**: Update `list_software.tex` immediately when adding new dependencies to `python/requirements.txt` or formatting macros in `CTUthesis.cls`.

## Fonts and PDF embedding

### Fonts in use
The class (`CTUthesis.cls`) loads:
- **Latin Modern** (`lmodern`, T1 encoding) — serif body text, sans-serif headings, monospaced URLs/code
- **AMS math fonts** (`amsfonts`, `amssymb`) — `msam`/`msbm` symbol sets for math

None of these are among the **14 standard PDF fonts** (Times, Helvetica, Courier, Symbol, ZapfDingbats and their Bold/Italic variants). Standard fonts are not usable here anyway — they lack Czech diacritics.

### Embedding
pdfLaTeX **automatically embeds all fonts** as subsetted Type1 (`.pfb`) outlines. No manual configuration needed. Verify after a build:

```
pdffonts latex/build/main.pdf
```

All entries should show `yes` in the `emb` column.

### Size impact
Subsetting embeds only the glyphs actually used, keeping overhead modest:

| Font group | Approx. embedded size |
|---|---|
| LM Roman (body text) | ~100–200 KB |
| LM Sans (headings) | ~50–100 KB |
| LM Mono (code/URLs) | ~30–80 KB |
| AMS symbol fonts | ~50–100 KB |
| **Total** | **~300–500 KB** |

For a 1–2 MB thesis PDF, fonts are the dominant component. There is no actionable saving here — embedding is mandatory for correct rendering on any system that does not have Latin Modern installed locally.
